# Test Coverage Guide

## Running Coverage Locally

```bash
# Run tests with coverage
python -m coverage run -m unittest discover -s tests -v

# View the report
python -m coverage report --show-missing

# Generate XML for CI/Codecov
python -m coverage xml

# Generate HTML report to browse in a browser
python -m coverage html
```

## Reading the Report

| Column | Meaning |
|--------|---------|
| `Name` | Module path |
| `Stmts` | Total executable statements |
| `Miss` | Statements not covered |
| `Cover` | Coverage percentage |
| `Missing` | Line numbers not hit during tests |

A line is "missing" if no test exercised that code path. Files at 100% are marked
with `COVERED` when `skip_covered = true` is set in `.coveragerc`.

## Adding Coverage to a Specific Module

1. Write a test in `tests/` that imports and exercises the module.
2. If the module has side effects on import, mock them or guard with
   `if __name__ == "__main__":` (already excluded).
3. Run `python -m coverage run -m unittest discover -s tests -v` and check
   that the new module appears in the report with improved numbers.

## Current Targets

| Metric | Value |
|--------|-------|
| Baseline (current) | 68% |
| Short-term goal | 75% |
| Long-term goal | 85%+ |

The CI pipeline enforces `--fail-under=68`. Any PR that drops below this will
fail. Raising the threshold is encouraged as coverage grows.

## Excluding Lines from Coverage

Some code is inherently untestable or intentionally unreachable. Mark these
lines so they don't drag down coverage:

```python
# pragma: no cover
def _debug_only():
    ...

# Also excluded automatically (see .coveragerc):
# - raise NotImplementedError
# - if __name__ == "__main__":
# - if TYPE_CHECKING:
```

Place `# pragma: no cover` on the line above the block or statement you want
to exclude.

## Configuration

Coverage settings live in `.coveragerc` at the repo root. Key settings:

- **`fail_under = 68`** — CI fails if coverage drops below this.
- **`show_missing = true`** — Shows uncovered line numbers.
- **`skip_covered = true`** — Hides fully-covered files for cleaner output.
- **`omit`** — Excludes test files, caches, and non-source directories.

## CI Enforcement

### How Coverage Is Enforced

The CI pipeline runs `scripts/enforce_coverage.py` after the test suite. This script:

1. Parses `coverage.xml` (generated by `coverage xml`).
2. Checks the overall coverage percentage against the configured threshold (68%).
3. Prints a formatted report showing every file's coverage.
4. Exits with code 1 if any file drops below the threshold.

The gate runs on every push and pull request. A failure blocks the merge.

### What Happens When Coverage Drops

- The CI job fails with a clear report listing under-performing files.
- The PR cannot be merged until coverage is brought back above 68%.
- Run `python -m coverage html` locally and open `htmlcov/index.html` to find
  which lines need tests.

### How to Add `# pragma: no cover`

For code that is genuinely untestable (debug helpers, dead branches, platform
guards), place the pragma on the line above the block:

```python
# pragma: no cover
def _unreachable_in_production():
    ...
```

Do **not** use pragma to skip writing tests for normal business logic. Use it
only for code that cannot be exercised in a test environment.

### How to View the HTML Report Locally

```bash
python -m coverage html
open htmlcov/index.html          # macOS
start htmlcov/index.html         # Windows
xdg-open htmlcov/index.html      # Linux
```

The report highlights uncovered lines in red and covered lines in green. Click
a file name to drill into line-level detail.

### How to Identify Uncovered Code

```bash
# Quick terminal summary with missing line numbers
python -m coverage report --show-missing

# Per-file breakdown via the enforcement script
python scripts/enforce_coverage.py --xml coverage.xml
```

Both show which lines were never executed during the test run. Focus on files
closest to the threshold first — small improvements there move the overall
percentage the most.
